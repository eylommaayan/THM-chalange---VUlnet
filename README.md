# THM-chalange---VUlnet




## שלב 1: סריקת פורטים וזיהוי שירותים (Service Enumeration)

### **תיאור (Description)**
השלב הראשון בתהליך הפריצה הוא זיהוי "שטח התקיפה" (Attack Surface) של מכונת היעד. לשם כך, בוצעה סריקה באמצעות כלי ה-**Nmap** כדי לזהות פורטים פתוחים, שירותים פעילים וגרסאות תוכנה הרצות עליהם.

### **ביצוע (Execution)**
הפקודה שהורצה:
`nmap 10.114.132.121 -sCV`

* **-sC**: הרצת סקריפטים כברירת מחדל של Nmap (לאיתור חולשות בסיסיות).
* **-sV**: זיהוי גרסאות השירותים (Version Detection).

### **ממצאים (Findings)**
הסריקה העלתה שני פורטים פתוחים עיקריים:

| פורט (Port) | מצב (State) | שירות (Service) | גרסה (Version) | הערות |
| :--- | :--- | :--- | :--- | :--- |
| **22** | Open | **SSH** | OpenSSH 7.6p1 | שירות לחיבור מרחוק (Ubuntu Linux). |
| **80** | Open | **HTTP** | Apache httpd 2.4.29 | שרת אינטרנט המציג אתר עם הכותרת "VulnNet". |

### **ניתוח (Analysis & Insights)**
1.  **פורט 80 (HTTP):** זהו וקטור התקיפה המרכזי בשלב זה. מכיוון שהסריקה זיהתה כותרת ספציפית ("VulnNet"), עולה החשד שהשרת עושה שימוש ב-Virtual Hosting, מה שדורש מאיתנו לבצע מיפוי DNS ידני (עריכת קובץ ה-Hosts) כדי לגשת לתוכן המלא.
2.  **פורט 22 (SSH):** גרסת ה-OpenSSH מעודכנת יחסית ואינה פגיעה לחולשות ידועות המאפשרות כניסה ללא פרטי גישה. לכן, פורט זה ישמש אותנו רק לאחר שנשיג שם משתמש וסיסמה/מפתח דרך האתר.

## שלב 3: סריקת ספריות (Directory Enumeration) - הכנת הכלים

### תיאור הפעולה (Description)
כדי לבצע סריקה אגרסיבית ויעילה של ספריות וקבצים חבויים, הוחלט להשתמש בכלי **Feroxbuster**. מכיוון שהכלי והמילונים המשלימים שלו לא היו קיימים במערכת ה-Kali, בוצעה התקנה מסודרת מהמאגרים הרשמיים.

### פקודות ביצוע (Execution)
```bash
sudo apt update && sudo apt install feroxbuster seclists -y
```

### רציונל (Rationale)
1. **Feroxbuster:** כלי שנכתב ב-Rust, המאפשר סריקה רקורסיבית במהירות גבוהה מאוד, מה שמקצר את זמן ה-Enumeration משמעותית.
2. **SecLists:** מאגר המילונים המקיף ביותר בתעשייה, הכולל רשימות ייעודיות ל-Discovery של ספריות, קבצים ותתי-דומיינים.

בצילום המסך רואים שסיימת להריץ את ה-**Feroxbuster** על הדומיין הראשי. נמצאו קבצים סטנדרטיים כמו `login.html`, `index.php` ותיקיות עיצוב (`css`, `js`, `img`), אבל כפי שצוין במדריך—**לא נמצא כאן וקטור פריצה ישיר.**

זה הרגע שבו פנטסטר מבין שהוא צריך להרחיב את החיפוש ולבדוק אם קיימים "סניפים" אחרים לשרת (תתי-דומיינים).

## שלב 4: סריקת ספריות וניתוח ממצאים ראשוני

### תיאור הפעולה (Description)
בוצע שימוש בכלי **Feroxbuster** לסריקה רקורסיבית של ספריות וקבצים תחת הדומיין `http://vulnnet.thm`. מטרת השלב הייתה לזהות דפי ניהול, קבצי הגדרה או ספריות חשופות שיכולים לשמש כנקודת כניסה.

### ממצאים (Findings)
הסריקה הניבה את התוצאות הבאות:
* **נתיבים סטטיים:** זוהו תיקיות `css/`, `js/`, ו-`img/`.
* **דפי קצה:** זוהו הקבצים `login.html` ו-`index.php`.
* **Directory Listing:** זוהתה אפשרות לצפייה בתוכן התיקיות (Directory Listing) בנתיבים מסוימים, אך לא נמצא בהן מידע רגיש או קבצי גיבוי.

### מסקנות ביניים (Interim Conclusion)
הסריקה על הדומיין הראשי לא העלתה חולשות קריטיות או קבצים המאפשרים הרצת קוד (RCE). כחלק ממתודולוגיית ה-Enumeration, יש לעבור לשלב הבא: **גילוי תתי-דומיינים (VHost Enumeration)**, כדי לבדוק אם קיימים שירותים נוספים המאוחסנים על אותו שרת תחת שמות דומיין שונים.
